Architecture

The package exposes the existing analytic-band column solvers through three levels of API.

High-level column update

radiative_heating! runs the reusable RadiativeTransferColumn workspace by calling longwave and shortwave component solvers. This is the convenience path for examples and single-column workflows.

Component access

Host models can call solve_longwave!, solve_shortwave!, and heating_rates! independently. This keeps solver, flux, and tendency ownership explicit for models that need their own vertical integrals or tendency insertion.

The staged interface also defines optical_properties!, cloud_optical_properties!, aerosol_optical_properties!, and radiative_fluxes! for future gas-optics and solver implementations. The first concrete staged solver is CloudlessLongwave, which consumes LongwaveOpticalProperties and writes caller-owned RadiativeFluxes. CloudlessShortwave does the same for absorptive shortwave optical depths stored in ShortwaveOpticalProperties.

heating_rates! can convert RadiativeFluxes into layer heating rates for a ColumnAtmosphere using explicit gravity and heat_capacity keywords. The convention is top-down pressure interfaces, net-downward flux, and positive heating for atmospheric warming.

Workspace access

radiation_workspace returns reusable storage for repeated runtime calls. For the current analytic-band path, RadiativeTransferColumn is already the workspace: it owns the temperature-tendency vector, shortwave transmissivity scratch, and diagnostics.

Future Breeze and SpeedyWeather integrations should prefer caller-owned arrays, views, or explicit workspaces over per-call temporary arrays.
